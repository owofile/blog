系统环境配置

每当我们装一个编程语言，比如JAVA，Python，首先需要的就是配置环境变量，C也是如此，首先我们需要下载MinGW-W64。

什么是MinGW-W6

MinGW-W64 就是“在 Windows 上免费用的 GCC 套装”，能把你的 .c / .cpp 文件直接编译成 Windows 下的 .exe 或 .dll。

  1. 它像一把瑞士军刀：里面最主要的工具叫 gcc/g++，负责把代码翻译成程序。
  2. 免费、小巧，装完就能在命令行里敲
    gcc hello.c -o hello.exe
    立刻得到可执行文件。
  3. 支持 32 位和 64 位，生成的程序在别人的 Windows 电脑上直接运行，不需要额外安装东西。
  4. 它只给“刀”——编译器、链接器、最基本的库。IDE、编辑器、调试图形界面、第三方库得自己另装。

总的来说，MinGW-W64就像是一个开源的编译工具链，提供了必要的工具，至于其他库则需要自己根据需求安装编译才能使用。

开始配置

在官网可以找到MinGW-W64的下载链接，也可以在其他下载网站下载，这里提供官网链接。

https://www.mingw-w64.org/

下载完成后再除了C盘以外的其他盘解压，进入到bin目录，复制路径。

打开系统变量，在path新建环境变量后，粘贴路径就好了。

完成后在cmd窗口通过查看版本命令检测是否安装成功

VSCODE配置

打开VSCODE搜索插件C/C++下载安装。

然后使用快捷键ctrl+shift+p打开VSCODE的搜索，输入c++,找到编辑配置（JSON）,进入到插件的配置设置界面。

在指定编译器路径中，复制我们之前下载好的MinGW里bin的gcc.exe文件的完整路径。

然后在IntelliSense模式中选择gcc-x64(legacy),即可设置完成。

需要注意的是，在同一个项目文件夹中，配置文件只会生成一次，如果新建立了其他项目文件夹，则需要重新生成。

如果你下一次生成的时候发现没有c_cpp文件，那你需要先重复上面的编辑配置（JSON）步骤，编辑完成后，会出现这个文件。
