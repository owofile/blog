前言

有时候我觉得现在手写笔记是浪费时间，实际体验agent一段时间后，我对这种新的交互方式感到兴奋，像是第一次玩电脑一样，我完全可以简单把自己学到的东西给agent，专门管理我的博客，编写更加精美专业的内容，甚至添加图片和表格，在此之前我觉得这很累。

但仔细想想还是算了，因为对我而言，博客的内容不是给别人看的，而是为了让自己复习，这些编写习惯让我复习的时候更加方便，同时编写的过程也能加深自己的理解和思考。

在这个时代，自身的思考是我们最强大的武器，因为我们已经从劳工变成了决策者，至少比起大多数焦虑的人而言，去尝试和思考总不是一件坏事，逃避思考没有意义。

增加Agent

如果你看了上一篇笔记，那么恭喜你配置好了你的Openclaw，而本文主要是学习更多的内容，让Agent为你所用

一个Agent在执行不同任务的时候可能会出现混乱，就如同公司一样，我们应该让专业的人干专业的事情，所以如果你需要多任务处理，我的建议是增加Agent，配置对应模型，编写人格Soul，Skills，让它成为这方面的专家。

运行后首先会创建新的工作空间,回车就好

然后会询问是否从main复制配置文件，一般选yes就可以

询问是否配置模型，如果需要更换模型就选Yes，否则就使用main的配置

是否选择聊天工具

是否选择配置私聊政策 (no)

是否给这个agent绑定当前的社交账号

全部配置完成后就创建成功了，如果之后修改也可以手动到配置文件里修改，让AI帮你改也可以，很方便

严格的安全策略（Skill 文件权限等）

在Web UI我们可以在代理中选择不同的agent，给他们不同的文件权限和skills，术有专攻，一个很好的安全策略可以避免AI无意损坏你的重要文件，当然我的Openclaw直接工作在备用电脑，所以我不在乎全部full了。

clawhub

养龙虾，养字主要体现在给Openclaw装不同的技能，让它成为你的得力助手，在clawhub中有2w多个开源skill,选择合适自己就好，同时注意安全问题

Agent的工作原理

其实这部分我在犹豫是否放在这里说，因为目前我还在学习agent的原理，放在之后的Agent-Hello笔记里或许更加合适，但Openclaw本质上就是借鉴了目前已有的技术，Agent和Skill都不是它发明的，但是它完成了营销和给大众提供了一项方便的出口，最早Agent是在命令行使用的，同样具有读写权限，但是不方便圈外人使用。

Agent 通过主循环持续运转（感知-推理-执行），结合心跳机制监控健康状态、防止死锁；根据用户命令进行分层决策（规划→执行→反思），调用Skill 系统扩展能力边界；权限系统分层管控，高危操作保留人工确认，最终实现可靠、安全、可扩展的自动化需求处理。

Agent 架构的 持续循环机制（Perception-Reasoning-Action Loop）导致 token 消耗显著增加：每次迭代需携带 历史上下文（多轮对话状态）、环境反馈（工具执行结果）及 中间推理过程（Chain-of-Thought），上下文窗口随循环深度线性膨胀，单次请求可能累积数万 token。
