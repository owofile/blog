Agent学习笔记-智能体与语言模型基础

Lain
2026-03-10
agent, ai, python

前言

最近对agent很感兴趣，作为一个生活早已经离不开Ai的人，agent的大名我早已如雷贯耳，只是这种“高科技”的产品，我感觉一直理我们很远，高额的token不是普通家庭能够承受的，第一次使用是今天的新年kimi的活动让我每天都能抽中agent名额，既然有了那就试试吧，抱着这样的态度我尝试后，让我惊讶，这已经接近于那些宣传广告说的，一句话创造应用程序，但我个人感觉对于目前而言真正的“一句话”还是很难，但已经完成了绝大多数繁琐的步骤，我越来越理解在文章上不断看到的那一句话

“AI的出现，不是代替程序员，而是让程序员化成为产品经理，真正从齿轮中解放，亦或者人人都可以成为产品经理”

然后我决定了解一下，在谷歌搜索中发现了Hello-Agents的开源项目(https://github.com/datawhalechina/Hello-Agents)，跟着教程开始学习，同时记录，也就有了这次的学习笔记%EF%BC%8C%E8%B7%9F%E7%9D%80%E6%95%99%E7%A8%8B%E5%BC%80%E5%A7%8B%E5%AD%A6%E4%B9%A0%EF%BC%8C%E5%90%8C%E6%97%B6%E8%AE%B0%E5%BD%95%EF%BC%8C%E4%B9%9F%E5%B0%B1%E6%9C%89%E4%BA%86%E8%BF%99%E6%AC%A1%E7%9A%84%E5%AD%A6%E4%B9%A0%E7%AC%94%E8%AE%B0)

什么是智能体

在探索任何一个复杂概念时，我们最好从一个简洁的定义开始。在人工智能领域，智能体被定义为任何能够通过传感器（Sensors）感知其所处环境（Environment），并自主地通过**执行器（Actuators）采取行动（Action）**以达成特定目标的实体。

Workflow和Agent的区别

目前主流有两种模式

Workflow 是让 AI 按部就班地执行指令，而 Agent 则是赋予 AI 自由度去自主达成目标

实践

最后我开始实践，文章给出一些代码创建一个只能旅游助手，使用python编写三个脚本协作

首先下载库

requests主要用于https请求，tavily-python则是专门让AI检索搜索引擎的库，需要注册配置API,最后是poenai，用于请求AI API

首先编写工具也就是SKII

Hello-Agents的用于查询天气服务的工具我无法使用，这里也是用tavily-python完成，最终实现两个工具类，查询城市天气和查询城市适合旅游景点

然后是中枢，llm_client.py用于调用LLM，我也自己改了一下，因为我使用的API不能频繁调用需要自动重试

最后就是核心代码，填写API和输入提示词，并且自动循环调用SKII，直到完成

整个逻辑其实并不复杂，为了防止AI对本机造成损坏，所以编写SKII和运行SKII本质上都要进行严格的规则审查，由于这次编写的SKII只是简单的请求所以规则不是太严格
